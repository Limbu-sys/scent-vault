## Что такое LEGAL_* и откуда взять значения

Это **ваши данные как продавца** (самозанятого). Они подставляются в оферту и политику конфиденциальности на сайте — по закону покупатель должен видеть, кто продаёт товар.

| Переменная | Что это | Где взять |
|------------|---------|-----------|
| `LEGAL_SELF_EMPLOYED_NAME` | ФИО полностью | Как в приложении **«Мой налог»** / паспорте |
| `LEGAL_INN` | ИНН (12 цифр) | «Мой налог» → Профиль, или [nalog.ru](https://service.nalog.ru/inn.do) |
| `LEGAL_CONTACT_EMAIL` | Email для покупателей | У вас уже: `dvetochkiinfo@gmail.com` |
| `LEGAL_PHONE` | Телефон | Ваш мобильный, например `+79001234567` |
| `LEGAL_ADDRESS` | Адрес продавца | Город или адрес регистрации, например `г. Краснодар` |

**Если вы ещё не самозанятый** — для тестового деплоя можно временно оставить пустым; перед реальными продажами нужно встать на учёт в «Мой налог» и заполнить ФИО + ИНН.

### Где заполнить на Amvera (после создания проекта)

1. Откройте проект в Amvera
2. Меню слева: **«Переменные»** или **«Настройки» → «Переменные окружения»**
3. Кнопка **«Добавить»** — имя переменной слева, значение справа
4. Сохранить → **перезапустить** / **Redeploy** проект

Локально то же самое в файле `scent-vault/.env` (он уже есть, не попадает в Git).

## Шаг 1. Репозиторий

1. Залейте папку `scent-vault` в Git (GitHub / GitLab)
2. В [Amvera](https://amvera.ru) → **Создать проект** → подключить репозиторий
3. Amvera подхватит `Dockerfile` и `amvera.yaml` из корня

## Шаг 2. Переменные окружения (Amvera → Переменные)

**Минимум для первого запуска** (остальное — после деплоя):

| Переменная | Значение |
|------------|----------|
| `TELEGRAM_BOT_TOKEN` | Токен от @BotFather |
| `TRIBUTE_API_KEY` | Ключ из Tribute → Settings → API Keys |
| `ADMIN_TELEGRAM_IDS` | `122429011` |
| `PRODUCTION_MODE` | `true` |
| `ALLOW_LOCAL_DEV` | `false` |
| `DATA_DIR` | `/data` |
| `PORT` | `80` |
| `WAREHOUSE_CITY` | `Краснодар` |
| `LEGAL_CONTACT_EMAIL` | `dvetochkiinfo@gmail.com` |
| `DGIS_API_KEY` | *(опционально)* ключ 2GIS для обновления базы поставщиков в админке |

**После первого деплоя** добавьте:

| Переменная | Значение |
|------------|----------|
| `APP_PUBLIC_URL` | `https://ИМЯ-ПРОЕКТА.amvera.io` |
| `TRIBUTE_SHOP_URL` | Ссылка на товар Tribute (см. TRIBUTE.md) |

**Юридические** (когда будете готовы к продажам):

| Переменная | Значение |
|------------|----------|
| `LEGAL_SELF_EMPLOYED_NAME` | Ваше ФИО |
| `LEGAL_INN` | ИНН самозанятого |
| `LEGAL_PHONE` | Телефон |
| `LEGAL_ADDRESS` | Адрес для оферты |

> **Важно:** после первого деплоя скопируйте URL проекта в `APP_PUBLIC_URL` и сделайте **Redeploy**.

## Шаг 3. Telegram-бот

Бот настраивается автоматически при старте контейнера (`bot.py`):

- Команда `/start` с кнопкой магазина
- **Menu Button** «🛍 Магазин» → `APP_PUBLIC_URL`

Дополнительно в @BotFather:

1. `/setdescription` — описание магазина
2. `/setuserpic` — загрузите `webapp/assets/icon-512.png`
3. Убедитесь, что домен Amvera добавлен: `/setdomain` → ваш `*.amvera.io`

Проверка: откройте бота → кнопка меню слева от поля ввода открывает Mini App.

## Шаг 4. Tribute webhook

Подробно: [TRIBUTE.md](./TRIBUTE.md)

Кратко:

1. Tribute → Settings → API Keys
2. **Webhook URL:** `https://ИМЯ-ПРОЕКТА.amvera.io/webhook/tribute`
3. API key = `TRIBUTE_API_KEY`
4. Создайте товар, ссылку → `TRIBUTE_SHOP_URL`

## Шаг 5. Хранилище

`amvera.yaml` монтирует `/data`:

- `scent_vault.db` — каталог, заказы
- `uploads/` — фото товаров

Данные сохраняются между деплоями.

## Шаг 6. Проверка после деплоя

- `https://ваш-url.amvera.io/api/health` → `{"ok":true}`
- Открыть магазин из Telegram
- Админка: `/admin` (ваш Telegram ID)
- Тестовый заказ → оплата Tribute с `#НОМЕР` в комментарии

## Локальный запуск

```powershell
cd scent-vault
pip install -r requirements.txt
copy config.example.env .env
# заполните .env
.\start.bat
```

- Магазин: http://127.0.0.1:8782
- Админка: http://127.0.0.1:8782/admin

## Юридические документы

Заполните `LEGAL_*` — данные подставятся в:

- `/legal/privacy` — 152-ФЗ
- `/legal/terms` — пользовательское соглашение
- `/legal/offer` — публичная оферта (обязательно для продаж в РФ)
